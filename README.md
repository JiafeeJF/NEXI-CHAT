# NEXI CHAT - 开源项目指引

> 最后更新：2026-03-07

## 项目简介
NEXI CHAT 是一个基于 Node.js 开发的多频道聊天应用，采用现代化的简约风格设计，支持实时消息传递、图片上传、语音消息等功能。

## 技术栈
- **后端**：Node.js + Express + Socket.io
- **前端**：HTML5 + CSS3 + JavaScript (ES6+)
- **数据库**：JSON 文件存储（轻量级）
- **实时通信**：Socket.io
- **安全**：BCrypt 密码加密、JWT 认证
- **文件处理**：Multer + Sharp

## 功能特性
- ✅ 多频道实时聊天
- ✅ 支持文本消息、图片上传、语音消息
- ✅ 用户认证与授权
- ✅ 管理员控制面板
- ✅ 消息回复功能

## 快速开始

### 克隆后直接运行（推荐）

从 GitHub 拉取后，在 **`nexichat` 目录**（即 `package.json` 所在目录）执行：

```bash
cd nexichat
npm install
npm run reset-data
npm start
```

浏览器访问 **https://localhost:3001** 或按下方「访问应用」使用前端地址。首次使用默认管理员：**admin** / **admin123**（可在 `.env` 中修改）。

若需同时启动前端静态服务，可执行 `npm run start-all`。

### 启动包（一键启动）

- **Windows**：双击 **`一键启动.bat`**（放在仓库根目录，与 `nexichat` 同级）。
- 启动器会：检查 Node 版本、按需安装依赖、同时启动后端与前端，就绪后自动打开浏览器。
- 关闭启动窗口或按 Ctrl+C 即可停止服务。

### 服务器一键部署与启动（Linux / 同目录脚本）

与 **`一键启动.bat`** 同目录下提供两个 Shell 脚本，用于在 Linux 服务器上自动部署与启动：

- **`一键部署启动.sh`**  
  - 自动检测并安装 Node.js 18+（支持 Debian/Ubuntu、CentOS/RHEL、Alpine，需 root/sudo）。  
  - 自动安装依赖（失败自动重试 3 次）。  
  - 单端口模式（`SERVE_FRONTEND=1`）：后端同端口提供前端，适合服务器部署。  
  - 用法：  
    - `./一键部署启动.sh` — 前台启动（端口默认 3000）  
    - `./一键部署启动.sh --background` — 后台启动，日志在 `nexichat/server/logs/server.log`  
    - `./一键部署启动.sh --install` — 仅安装依赖  
    - `./一键部署启动.sh --port 8080` — 指定端口  

- **`停止.sh`**  
  - 停止由「一键部署启动.sh」后台启动的服务；若无 PID 文件则尝试按进程清理残留。  

首次使用需赋予执行权限：`chmod +x 一键部署启动.sh 停止.sh`。

### 1. 环境要求
- Node.js 16.0 或更高版本（推荐 18+；使用 `一键部署启动.sh` 时可自动安装）
- npm 8.0 或更高版本

### 2. 安装依赖
在 **nexichat** 目录下执行：
```bash
npm install
```

### 3. 配置项目

#### 创建环境变量文件（可选）
在 **nexichat** 目录下创建 `.env` 文件（可复制 `nexichat/.env.example` 为 `.env`），配置以下参数：
```
PORT=3000
JWT_SECRET=your-secret-key-change-in-production
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123
```

#### 配置 SSL 证书
1. 在 `cert/` 目录下放置 SSL 证书文件（cert.crt）和私钥文件（cert.key）
2. 或运行以下命令生成自签名证书：
```bash
npm run generate-cert
```

### 4. 启动项目

#### 启动主服务器（HTTPS）
```bash
npm start
```

#### 启动前端服务器（HTTP）
```bash
npm run frontend
```

#### 同时启动两个服务器
```bash
npm run start-all
```

### 5. 清空数据（可选）
克隆后首次运行或需要恢复初始状态时，在 **nexichat** 目录执行：
```bash
npm run reset-data
```
将清空所有用户数据、消息、好友、上传文件，仅保留默认管理员（用户名 `admin`，密码 `admin123`）。**执行后需重启服务。**

### 6. 访问应用
- 后端：https://localhost:3000
- 前端应用：http://localhost:3001

## 使用说明

### 用户注册与登录
1. 访问登录页面：https://localhost:3001/login.html
2. 点击"注册"链接创建新账号
3. 使用创建的账号登录

### 管理员功能
1. 访问管理员登录：https://localhost:3001/admin-login.html（或同端口下的 /admin-login.html）
2. 使用管理员凭证登录（默认：admin/admin123）
3. 管理员可以：管理用户、封禁/解封、删除用户；管理频道；查看日志；**处理账号注销申请**（列表查看、点击「查看」弹窗看理由、通过/拒绝，新申请与状态变更实时刷新）

### 发送消息与回复/撤回
1. 选择左侧频道或切换到「私聊」并选择好友
2. 在消息输入框中输入内容，点击发送按钮或按 Enter 键发送
3. **回复**：右键某条消息气泡，在气泡正上方出现的横向选择条中点击「回复」，输入内容后发送即可回复该条
4. **撤回**：仅限自己发送的消息、2 分钟内可撤回。右键该消息气泡，在选择条中点击「撤回」；撤回后该条消失，中间显示「你撤回了一条消息 重新编辑」（可点击「重新编辑」把原文填回输入框）；撤回后 5 分钟该条会从数据库删除并移除界面提示（实时生效，无需刷新）

### 私聊安全
私聊消息内容在服务端以 **AES-256-GCM** 加密后落库，防止数据文件泄露时被直接查看。生产环境请在 `.env` 中设置 `DM_ENCRYPTION_KEY`（32 字节 hex 或足够长字符串）；未设置时使用开发默认密钥。

### 上传图片
1. 点击图片上传按钮
2. 选择要上传的图片文件
3. 等待上传完成后自动发送

### 发送语音消息
1. 按住麦克风按钮开始录制
2. 松开按钮结束录制并发送
3. 最长录制时间为 60 秒

### 好友与私聊
1. 点击左侧「添加好友」按钮，输入对方**用户名**发送好友申请（不支持通过 ID 添加）
2. 在「好友与私聊」弹窗中可查看待处理的好友申请并同意/拒绝
3. 切换到「私聊」模式，从左侧好友列表选择好友即可进行一对一私聊（支持文字、图片、语音及回复、撤回）

### 账号注销申请
1. 在「个人设置」底部找到「账号注销」区块，点击「申请注销账号」
2. 在弹窗中**必填**注销理由（最多 200 字）后提交；每人每日最多 3 次申请，超限后锁定 24 小时
3. 管理员在后台「注销申请」中处理；通过后账号将被删除，拒绝后可再次申请
4. 申请与审核结果会实时推送到用户端与管理端

## 项目结构
```
nexichat/
├── cert/                 # SSL 证书文件
├── public/               # 前端静态资源
│   ├── aud/              # 音频文件
│   ├── css/              # 样式文件
│   ├── images/           # 图片资源
│   ├── js/               # JavaScript 文件
│   ├── utils/            # 前端工具（如 B 站视频嵌入）
│   ├── uploads/          # 上传文件存储
│   └── videos/           # 视频文件
├── server/               # 后端代码
│   ├── data/             # 数据库文件（用户、频道消息、好友、好友申请、私聊消息、注销申请及锁定等）
│   ├── logs/             # 日志文件
│   ├── badwords.js       # 屏蔽词过滤逻辑
│   ├── badwords.json     # 屏蔽词词库
│   ├── db.js             # 数据库操作
│   └── log.js            # 日志系统
├── scripts/              # 脚本与前端服务
│   ├── generate-cert.js  # 证书生成工具
│   ├── frontend-server.js # 前端服务器
│   └── check-cert.js     # 证书检查工具
├── index.js              # 主服务器入口
├── package.json          # 项目配置
└── package-lock.json     # 依赖锁文件
```

## 提交到 GitHub 时注意（避开敏感资源）

项目已通过 **`.gitignore`** 排除以下内容，避免敏感或运行时资源被提交：

- **环境配置**：`.env`（仅保留 `.env.example` 供克隆后复制使用）
- **数据与日志**：`nexichat/server/data/`、`nexichat/server/logs/`
- **用户上传**：`nexichat/public/uploads/`、`nexichat/public/videos/`
- **证书与私钥**：`nexichat/cert/`
- **依赖**：`node_modules/`

若此前曾提交过上述目录，可在仓库根目录执行以下命令从 Git 中移除（不删本地文件）：

```bash
git rm -r --cached nexichat/server/data nexichat/server/logs nexichat/public/uploads 2>/dev/null; true
git commit -m "chore: 从版本库移除敏感/运行时目录"
```

克隆后首次运行会自动创建 `server/data`、`server/logs` 及所需 JSON 文件。

## 安全注意事项
- 生产环境中请务必修改默认的 JWT 密钥
- 定期更新依赖包以修复安全漏洞
- 不要将敏感信息（如证书、密钥）提交到版本控制系统
- 使用 HTTPS 协议保护数据传输
