# NEXI CHAT - 开源项目指引

## 项目简介
NEXI CHAT 是一个基于 Node.js 开发的多频道聊天应用，采用现代化的简约风格设计，支持实时消息传递、图片上传、语音消息等功能。

## 技术栈
- **后端**：Node.js + Express + Socket.io
- **前端**：HTML5 + CSS3 + JavaScript (ES6+)
- **数据库**：JSON 文件存储（轻量级）
- **实时通信**：Socket.io
- **安全**：BCrypt 密码加密、JWT 认证
- **文件处理**：Multer + Sharp

## 功能特性
- ✅ 多频道实时聊天
- ✅ 支持文本消息、图片上传、语音消息
- ✅ 用户认证与授权
- ✅ 管理员控制面板
- ✅ 消息回复功能
- ✅ 消息撤回功能（2分钟内）
- ✅ 屏蔽词过滤
- ✅ 私聊频道（带密码保护）
- ✅ 好友系统（添加好友、好友申请、同意/拒绝、删除好友）
- ✅ 好友私聊（与好友一对一私聊，支持文字/图片/语音；私聊消息支持回复与 2 分钟内撤回）
- ✅ 频道与私聊的清除时间（按用户记录“已清除”时间，用于未读/历史展示）
- ✅ 聊天中的 B 站视频链接解析与嵌入播放

## 快速开始

### 克隆后直接运行（推荐）

从 GitHub 拉取后，在 **`nexichat` 目录**（即 `package.json` 所在目录）执行：

```bash
cd nexichat
npm install
npm run reset-data
npm start
```

浏览器访问 **https://localhost:3000** 或按下方「访问应用」使用前端地址。首次使用默认管理员：**admin** / **admin123**（可在 `.env` 中修改）。

若需同时启动前端静态服务，可执行 `npm run start-all`。

### 专业启动包（一键启动）

- **Windows**：双击 **`一键启动.bat`**（可放在仓库根目录或 `nexichat` 目录下）。
- 启动器会：检查 Node 版本、按需安装依赖、同时启动后端与前端，就绪后自动打开浏览器。
- 关闭启动窗口或按 Ctrl+C 即可停止服务。

---

### 1. 环境要求
- Node.js 16.0 或更高版本
- npm 8.0 或更高版本

### 2. 安装依赖
在 **nexichat** 目录下执行：
```bash
npm install
```

### 3. 配置项目

#### 创建环境变量文件（可选）
在 **nexichat** 目录下创建 `.env` 文件（可复制 `nexichat/.env.example` 为 `.env`），配置以下参数：
```
PORT=3000
JWT_SECRET=your-secret-key-change-in-production
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123
```

#### 配置 SSL 证书
1. 在 `cert/` 目录下放置 SSL 证书文件（cert.crt）和私钥文件（cert.key）
2. 或运行以下命令生成自签名证书：
```bash
npm run generate-cert
```

### 4. 启动项目

#### 启动主服务器（HTTPS）
```bash
npm start
```

#### 启动前端服务器（HTTP）
```bash
npm run frontend
```

#### 同时启动两个服务器
```bash
npm run start-all
```

### 5. 清空数据（可选）
克隆后首次运行或需要恢复初始状态时，在 **nexichat** 目录执行：
```bash
npm run reset-data
```
将清空所有用户数据、消息、好友、上传文件，仅保留默认管理员（用户名 `admin`，密码 `admin123`）。**执行后需重启服务。**

### 6. 访问应用
- 后端：https://localhost:3000
- 前端应用：http://localhost:3001

## 使用说明

### 用户注册与登录
1. 访问登录页面：https://localhost:3001/login.html
2. 点击"注册"链接创建新账号
3. 使用创建的账号登录

### 管理员功能
1. 访问管理员登录：https://localhost:3001/admin-login.html
2. 使用管理员凭证登录（默认：admin/admin123）
3. 管理员可以管理用户、频道和查看日志

### 发送消息
1. 选择左侧频道或切换到「私聊」并选择好友
2. 在消息输入框中输入内容
3. 点击发送按钮或按 Enter 键发送

### 私聊安全
私聊消息内容在服务端以 **AES-256-GCM** 加密后落库，防止数据文件泄露时被直接查看。生产环境请在 `.env` 中设置 `DM_ENCRYPTION_KEY`（32 字节 hex 或足够长字符串）；未设置时使用开发默认密钥。

### 上传图片
1. 点击图片上传按钮
2. 选择要上传的图片文件
3. 等待上传完成后自动发送

### 发送语音消息
1. 按住麦克风按钮开始录制
2. 松开按钮结束录制并发送
3. 最长录制时间为 60 秒

### 好友与私聊
1. 点击左侧「添加好友」按钮，输入对方用户名或 ID 发送好友申请
2. 在「好友与私聊」弹窗中可查看待处理的好友申请并同意/拒绝
3. 切换到「私聊」模式，从左侧好友列表选择好友即可进行一对一私聊（支持文字、图片、语音及回复、撤回）

## 项目结构
```
nexichat/
├── cert/                 # SSL 证书文件
├── public/               # 前端静态资源
│   ├── aud/              # 音频文件
│   ├── css/              # 样式文件
│   ├── images/           # 图片资源
│   ├── js/               # JavaScript 文件
│   ├── utils/            # 前端工具（如 B 站视频嵌入）
│   ├── uploads/          # 上传文件存储
│   └── videos/           # 视频文件
├── server/               # 后端代码
│   ├── data/             # 数据库文件（含频道消息、用户、好友、好友申请、私聊消息等）
│   ├── logs/             # 日志文件
│   ├── badwords.js       # 屏蔽词过滤逻辑
│   ├── badwords.json     # 屏蔽词词库
│   ├── db.js             # 数据库操作
│   └── log.js            # 日志系统
├── scripts/              # 脚本与前端服务
│   ├── generate-cert.js  # 证书生成工具
│   ├── frontend-server.js # 前端服务器
│   └── check-cert.js     # 证书检查工具
├── index.js              # 主服务器入口
├── package.json          # 项目配置
└── package-lock.json     # 依赖锁文件
```

## 安全注意事项
- 生产环境中请务必修改默认的 JWT 密钥
- 定期更新依赖包以修复安全漏洞
- 不要将敏感信息（如证书、密钥）提交到版本控制系统
- 使用 HTTPS 协议保护数据传输
